What is AuthChain?

AuthChain is an extension to the basic HTTP scheme for browsing websites. It works on top of the default HTTP behavior to provide extra user agent authentication methods. It is designed to degrade gracefully if a user agent does not support the AuthChain protocol extension.

AuthChain aims to provide a unified protocol for negotiating the following types of authentication methods:

  • Simple page-based authentication (e.g. HTTP POST forms)
  • HTTP Authorization (HTTP 401)
  • Non-standard authentication scheme

Each of the above potentially has different underlying transport mechanisms, handshake protocols, failover handling procedures, user agent compatibility matrices, design limitations, and usability issues. AuthChain does not try to account for discrepancies between them; it outlines a common HTTP-based interface between a user agent and a server-side authentication service in a web-browsing context.
